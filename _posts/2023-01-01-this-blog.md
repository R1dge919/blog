　このブログについてというよりは、大半は「GitHub Pages」と「Jekyll」を使ったブログ作成について。大雑把に説明を残しておく。

## GitHub Pages
　リポジトリを作成し、HTMLファイルなどを配置、
- Settings
    - Pages
        - Build and deployment
            - Branch

から、使用するブランチと、ルートディレクトリを指定することで、リポジトリを基にした静的サイト`https://<ユーザー名>.github.io/<リポジトリ名>`が作成される。  
※リポジトリ名を`<ユーザー名>.github.io`にすると、WebサイトのURLは`https://<ユーザー名>.github.io`となる。

## Jekyll
　プレーンテキストを静的サイトに変えるジェネレーター。言い換えると、Markdownを勝手にWebページに変換する。  
　GitHub Pagesはこのジェネレーターに対応しているため、適切な設定ファイルの記述などを行うと、GitHub側が自動でプッシュされたデータを基にWebページの生成を行う。  
（GitHub Actionsとやらを使えば、Jekyllでない他のジェネレーターも使えるらしい）  

　**このため、Jekyllで構成されたページのテンプレートリポジトリをクローンすれば、あとはMarkdownを指定の位置に配置してリポジトリにプッシュ→デプロイを待つだけで、自分のWebページが完成！**  
　テンプレートについては、[Jekyllのドキュメント](https://jekyllrb.com/docs/themes/)内にいくつか纏められたWebサイトのリンクが貼られています。  
（本ブログは[Gesko](https://github.com/DavideBri/Gesko)を使用）
